## Authentication

### Use Laravel's Built-In Auth Scaffolding

```bash
composer require laravel/ui
php artisan ui bootstrap --auth   # or vue/react
php artisan migrate
```

This generates Login, Register, Password Reset, and Email Verification flows.

### Protecting Routes

```php
// Middleware on routes
Route::middleware('auth')->group(function () {
    Route::get('/dashboard', 'DashboardController@index');
});

// Or in controller constructor
public function __construct()
{
    $this->middleware('auth');
    $this->middleware('verified'); // requires email verification
}
```

### Getting the Authenticated User

```php
// In controllers/middleware
$user = $request->user();
$user = Auth::user();
$id   = Auth::id();

// Checking authentication
if (Auth::check()) { /* logged in */ }
if (Auth::guest()) { /* not logged in */ }
```

### Guards — Use When You Have Multiple User Types

```php
// In config/auth.php, define guards per user type
// API guard for API authentication
// Admin guard for separate admin users

Auth::guard('admin')->user()
$request->user('admin')
```

## Authorization — Gates and Policies

### Prefer Policies Over Gates for Model-Based Authorization

```bash
php artisan make:policy PostPolicy --model=Post
```

```php
class PostPolicy
{
    public function view(User $user, Post $post): bool
    {
        return true; // everyone can view published posts
    }

    public function update(User $user, Post $post): bool
    {
        return $user->id === $post->user_id;
    }

    public function delete(User $user, Post $post): bool
    {
        return $user->id === $post->user_id || $user->isAdmin();
    }
}
```

Register in `AuthServiceProvider`:

```php
protected $policies = [
    Post::class => PostPolicy::class,
];
```

Use in controllers:

```php
public function update(UpdatePostRequest $request, Post $post)
{
    $this->authorize('update', $post);  // throws 403 if fails
    // ...
}
```

Use in Blade:

```blade
@can('update', $post)
    <a href="{{ route('posts.edit', $post) }}">Edit</a>
@endcan

@cannot('delete', $post)
    <p>You cannot delete this post.</p>
@endcannot
```

### Gates — For Non-Model Authorization

```php
// In AuthServiceProvider::boot()
Gate::define('access-admin', function (User $user) {
    return $user->isAdmin();
});

// Usage
if (Gate::allows('access-admin')) { /* ... */ }
Gate::authorize('access-admin'); // throws 403 if denied
```

## CSRF Protection

- All POST/PUT/PATCH/DELETE web routes require a CSRF token
- Always include `@csrf` in Blade forms

```blade
<form method="POST" action="{{ route('posts.store') }}">
    @csrf
    {{-- form fields --}}
</form>

{{-- Method spoofing for PUT/DELETE --}}
<form method="POST" action="{{ route('posts.update', $post) }}">
    @csrf
    @method('PUT')
    {{-- form fields --}}
</form>
```

For AJAX requests, include the token in the header:

```javascript
// Set globally with Axios (already done in bootstrap.js scaffolding)
window.axios.defaults.headers.common['X-CSRF-TOKEN'] =
    document.querySelector('meta[name="csrf-token"]').getAttribute('content');
```

Exclude routes from CSRF (e.g. webhooks) in `VerifyCsrfToken` middleware:

```php
protected $except = [
    'stripe/*',
    'webhooks/github',
];
```

## Password Hashing

**Never store plain-text passwords. Always use Bcrypt (default) or Argon2.**

```php
// Hashing a password
$hashed = Hash::make($request->password);
// or
$hashed = bcrypt($request->password);

// Verifying a password
if (Hash::check($plaintext, $hashed)) {
    // correct password
}

// Rehash when needed (e.g. cost factor changed)
if (Hash::needsRehash($hashed)) {
    $user->password = Hash::make($plaintext);
    $user->save();
}
```

Configuration in `config/hashing.php`:

```php
'driver' => 'bcrypt',
'bcrypt' => ['rounds' => 10],       // 10-12 recommended
'argon'  => ['memory' => 1024, 'threads' => 2, 'time' => 2],
```

## Encryption

Use Laravel's built-in encryption (AES-256-CBC) — never roll your own:

```php
// Encrypt
$encrypted = encrypt($value);
$encrypted = Crypt::encryptString($string); // returns base64 string

// Decrypt
$decrypted = decrypt($encrypted);
$decrypted = Crypt::decryptString($encrypted);
```

Make sure `APP_KEY` is set in `.env` (32-character random string, generated by `php artisan key:generate`).

## API Authentication — Sanctum (Recommended for SPA / Mobile)

```bash
composer require laravel/sanctum
php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
php artisan migrate
```

```php
// Issue token
$token = $user->createToken('mobile-app')->plainTextToken;

// Protect routes
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/user', function (Request $request) {
        return $request->user();
    });
});
```

Use **Passport** for full OAuth2 server requirements (third-party apps, authorization codes).

## Security Checklist

- [ ] `APP_DEBUG=false` in production
- [ ] `APP_ENV=production` in production
- [ ] `APP_KEY` is set and unique per environment
- [ ] All passwords hashed with `Hash::make()` / `bcrypt()`
- [ ] CSRF protection enabled on all web forms (`@csrf`)
- [ ] Mass-assignment protected (`$fillable` or `$guarded`)
- [ ] User input never trusted raw in DB queries
- [ ] Authorization checked in controllers (`$this->authorize()`)
- [ ] Sensitive attributes in `$hidden`
- [ ] HTTPS enforced in production (`url()->forceScheme('https')` or server config)
